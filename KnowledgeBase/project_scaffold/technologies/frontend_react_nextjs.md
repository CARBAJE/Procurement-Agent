# Frontend — React 18 / Next.js 14

> [!architecture] Role in the System
> The frontend layer is the primary interaction surface between enterprise procurement users and the [[beckn_bap_client|Beckn BAP Agent]]. Built with **React 18** and **Next.js 14**, it renders the procurement dashboard, comparison views, approval interfaces, and real-time order tracking. All requests flow through the [[api_gateway]] before reaching the agent backend.

## Stack

| Technology | Version | Role |
|---|---|---|
| React | 18 | Core UI library |
| Next.js | 14 | SSR framework |
| TypeScript | latest | Type-safe development |
| Tailwind CSS | latest | Utility-first styling |
| ShadCN UI | latest | Component library |

> [!tech-stack] Why This Stack
> - **Next.js 14 with SSR** delivers enterprise-grade performance — pages render server-side, reducing time-to-first-meaningful-paint on procurement dashboards that load comparison tables with many rows.
> - **TypeScript** enforces type safety across all API contracts (Beckn intent schema, ERP response models), reducing integration bugs at compile time rather than runtime.
> - **ShadCN UI** provides accessible, composable components that match enterprise UI standards without vendor lock-in.

## Phase Delivery

> [!milestone] Phase 1 — Weeks 1–4 (Foundation Scaffold)
> - React/Next.js app running locally with SSO stub from [[identity_access_keycloak|Keycloak]].
> - Basic procurement request submission form functional.
> - **Acceptance criterion:** Request submission triggers [[nl_intent_parser]] pipeline end-to-end.

> [!milestone] Phase 2 — Weeks 5–8 (Comparison UI + Real-Time Tracking)
> - Side-by-side offer comparison view with [[comparison_scoring_engine|agent reasoning]] displayed.
> - [[real_time_tracking|Real-time order tracking]] via WebSockets — dashboard reflects status within 30 seconds of change.
> - [[approval_workflow|Approval interface]]: one-click approve/reject for approvers receiving routed orders.

> [!milestone] Phase 3 — Weeks 9–12 (Analytics Dashboard)
> - Full [[analytics_dashboard]] built using [[data_visualization|Recharts/D3]].
> - Spend analysis, savings tracking, supplier metrics — 6+ metrics with drill-down capability.

## Key Features Implemented

- **Natural language input** — free-form text procurement request box (or [[communication_slack_teams|Slack]] integration).
- **Side-by-side seller comparison** — explainability view from [[comparison_scoring_engine]].
- **Approval workflow UI** — one-click approve/reject for orders routed by [[approval_workflow]].
- **Real-time order status** — WebSocket-driven dashboard connected to [[real_time_tracking|/status polling]].
- **Analytics & benchmarking** — reports generated by [[analytics_dashboard]] for CPO use cases.
- **Conversational interface option** — users interact via [[communication_slack_teams|Slack/Teams]] directly.

> [!guardrail] SSO Enforcement
> Authentication is mandatory. The frontend does **not** render any procurement functionality until a valid session from [[identity_access_keycloak|Keycloak]] (SAML 2.0 / OIDC) is confirmed. The [[api_gateway]] enforces this at the API boundary — a requester cannot bypass [[approval_workflow|approval thresholds]] through direct API manipulation.
